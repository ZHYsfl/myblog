# AI 时代的后端双雄：Python 与 Go 如何在生态错位中实现“弱点消失”

python和go是我认为在AI时代，只会越来越火越吃香，市场额越来越多的两大后端语言。各自的原因值得好好分析一下。
首先，先说python为什么吃香了。python最大的缺点是GIL锁，这很大程度上是因为python其实“年纪”很大了，当时多核CPU并不常见，所以在当时设计的内存管理机制使得python不能使用多核CPU的能力，让python的并行其实是“假并行”。python的优点是语法简单，开发效率高，也适合快速开发一些小工具、小脚本，俗话说“人生苦短，我用python。那么AI时代，第一，开发迭代速度很重要，和python的优点契合；第二，AI 框架的 API 设计天然适合 Python 的动态特性；第三，训练模型的时间很长，而计算发生在GPU，对于cpu而言是IO等待操作，整个训练过程主要是GPU长达数小时到几个月的计算时间，和这么长的时间相比python的假并行简直是足球场上的一只蚂蚁。这最大程度规避了python的弱点，或者说，python在训练模型的时候，这个弱点已经到了消失的地步。总结来说，模型训练更像是python把计算任务外包给了C++和GPU，时间大头全在外包侧，自己的假并行根本不算什么。由于在AI界扬长避短的特性，python建立了强大的AI生态，这使得python更加不可一世，形成了不断巩固自己地位的强大正向循环。
再来说说go为什么吃香了。go语言最被人诟病的一点是if err!=nil 要写好多次，整个项目有4成5成在处理if err!=nil的时候，我们就会感觉枯燥，写得很痛苦。go语言的优点是内置优秀的并发模型，语法简陋（褒义），开发效率高，上手快，是典型的团队型语言，对程序员放权恰到好处。在AI时代，if err!=nil这种虽然不难，但原来需要一个个手敲的代码，现在可以通过Tab或者AI生成来直接写了，这就规避了go最大的缺点，而且go语言因为有效放权和语法简陋，且编译非常严格，AI 写错了编译器会报错，这两点使得AI写得代码错误率极低。go语言讨厌喜欢“秀”的程序员，所以在语言端就禁止了这种“秀”的权利的下放，我们写出来的代码即文档，AI 理解成本低。此外，go语言是典型的易读语言，AI写后我们人做review看得非常舒服。go语言最强大的并发模型，使得在云端基础设施完全是go的天下，而AI时代对云服务的需求大增，使得go语言某种意义上成为了AI时代云基础设施的底层语言。AI时代，go语言也能做到扬长避短，所以未来大有可为。
综合对比python和go，python语言相对go语言，是易写不易读语言（python虽然哲学是显式大于隐式，但是还是直接except exception as e处理的多，这点不如go易读），go语言相对python语言，是易读不易写语言。（但其实两个语言对比其他语言来说都算是易读易写了）。
总结一下，两个语言对比其他语言都算是易读易写，且两者在AI时代生态位重叠较少，应用场景不同（python更多是算法端，训练端；go更多是云端和正常企业级项目后端），都能做到扬长避短，所以两个语言在AI时代才能越来越火。
